#Track 2: FreeMoCap Ecosystem / Skelly Ecosystem

## Short summary
*Briefly describe the purpose of the proposal and the software project(s) it involves. 3,000 characters maximum* 

High-resolution 3D motion capture has traditionally required $50K–$200K+ in specialized hardware, dedicated lab space, and trained operators, limiting its use outside well-resourced laboratories. FreeMoCap is an open-source markerless motion-capture application, maintained by The FreeMoCap Foundation, that makes research-grade 3D movement data accessible using synchronized consumer cameras and open software. More than 12,000 users across ~140 countries have used the system, and independent research groups have published work spanning gait mechanics, reaching kinematics, clinical assessment, and computational linguistics.

FreeMoCap demonstrates demand for accessible camera-based movement analysis, but it is also evidence of a broader infrastructure need. The reusable capabilities beneath the software (i.e., synchronized video acquisition, camera calibration, tracking integration, 3D reconstruction, and structured movement-data representation) are the same capabilities needed by researchers turning camera footage into calibrated 3D measurements across the life sciences. Today, many communities rebuild this infrastructure from scratch, producing one-off code that is difficult to reuse, validate, or share.

This proposal seeks funding to develop the Skelly ecosystem: a set of composable open-source Python libraries that make these shared capabilities independently available as building blocks for camera-based 3D measurement. The ecosystem includes SkellyCam for synchronized multi-camera acquisition and recording metadata, SkellyTracker for connecting tracking outputs through standardized interfaces, SkellyForge for camera calibration and 3D reconstruction, and SkellyModels for structured skeletal and keypoint data with embedded metadata and quality metrics. Each library can be installed and used on its own or composed with others into pipelines tailored to a researcher’s specific domain and methods.

The proposed work spans four priorities: stabilizing and documenting public APIs so researchers can depend on each package and contributors can extend them through clear protocols; defining shared data schemas and quality metrics between pipeline stages so outputs from any component flow consistently into the next regardless of tracking method, camera setup, or analysis workflow; building programmatic, command-line, and batch-processing interfaces for large datasets, scripted analyses, and agentic research workflows beyond interactive GUI use; and releasing benchmarking tools that let researchers evaluate reconstruction quality, compare pipeline configurations, and determine whether outputs suit their scientific use case. Ultimately, we aim to give researchers who need camera-based 3D measurement access to proven shared infrastructure and ensure that when one community improves a component, that improvement becomes available to others.


## Expected value
*If the proposal is successfully funded, what does success look like? If successful, this proposal will make the Skelly ecosystem a reusable foundation for camera-based 3D movement analysis across the life sciences. Success means researchers can move from synchronized videos to structured, reproducible 3D datasets using shared open-source components rather than rebuilding acquisition, calibration, reconstruction, and data-organization infrastructure for each new project. Researchers will be able to use Skelly libraries individually, within FreeMoCap, or as composable parts of custom pipelines tailored to their research context.

The project will improve upstream software by giving camera-acquisition tools, tracking methods, calibration routines, and reconstruction algorithms shared interfaces for exchanging data. It will improve downstream software by producing standardized outputs consumable by biomechanics analyses, behavioral pipelines, visualization tools, statistical workflows, and machine-learning systems.

The work supports AI enablement and large-scale analysis by providing the surrounding infrastructure that camera-based AI workflows require: machine-readable schemas, provenance records, quality metrics, batch execution, and validated reconstruction pipelines. The result is an ecosystem where improvements to any stage of camera-based movement analysis propagate across projects and research communities instead of remaining trapped in one-off lab-specific code.


## Landscape Analysis 
*We are looking for proposals from software projects with demonstrated traction and adoption. Briefly describe other software tools that the audience for this proposal primarily uses (including proprietary alternatives, if they exist), and how the software project(s) in your proposal compare in terms of user base, adoption, functionality, and maturity relative to their target audience. You can add indicators of adoption and usage as needed. Please indicate if the software is used in AI applications and workflows. (1,500 characters maximum)*

Camera-based measurement in the life sciences spans distinct communities, including human motion capture (biomechanics, rehabilitation, neuroscience) and animal behavior tracking (neuroscience, ecology). Each has largely developed tools in isolation.

In human motion capture, Theia3D is proprietary (~$15K–$30K, closed-source); OpenCap is open-source but locked to one pose-estimation backend and iOS hardware; In animal behavior, DeepLabCut and SLEAP provide pose estimation but require researchers to build their own capture, calibration, and reconstruction infrastructure. No existing ecosystem provides shared, composable libraries for the full pipeline across these domains.

The Skelly ecosystem provides independently installable components for each pipeline stage with standardized interfaces between them. Adoption of FreeMoCap (10,000+ users, ~140 countries, ~7,400 GitHub stars, 3,000-member community, independent publications across gait, kinematics, clinical assessment, and computational linguistics) demonstrates traction for the integrated system. The proposed work makes each library independently robust and documented for use in researcher-built pipelines and AI-driven workflows.
