#Track 2: FreeMoCap Ecosystem / Skelly Ecosystem

## Short summary
*Briefly describe the purpose of the proposal and the software project(s) it involves. 3,000 characters maximum* 

Camera-based measurement is increasingly central to life-sciences research, from human motion capture in biomechanics, rehabilitation, and neuroscience to animal behavior tracking in ecology and comparative biology. These workflows often follow the same basic path: record synchronized video from multiple cameras, calibrate the cameras so their views share a common 3D coordinate system, identify points of interest in each image using AI-based pose estimation or other tracking methods, reconstruct those points in 3D, and organize the resulting movement data for analysis. Yet researchers routinely rebuild these foundational components from scratch. This repeated reinvention consumes effort that could otherwise support scientific discovery and often produces one-off code that is difficult to reuse, validate, automate, or maintain.

This proposal seeks funding for the coordinated development of the Skelly ecosystem: a set of composable open-source libraries for camera-based 3D movement analysis. These libraries power FreeMoCap, a free, open-source markerless motion-capture platform that generates research-grade 3D movement data from consumer cameras, while making the underlying capabilities available as modular tools for custom research pipelines. The proposed work spans multiple related open-source projects and focuses on the shared interfaces, data standards, and interoperability needed to make these tools usable independently or as parts of larger automated workflows.

The ecosystem includes SkellyCam for synchronized multi-camera acquisition and recording metadata, SkellyTracker for standardized interfaces to interchangeable AI pose-estimation models, SkellyForge for camera calibration and 3D reconstruction, and SkellyModels for shared skeletal and keypoint data structures. Each package can be installed standalone; together, they compose into pipelines for human gait analysis, balance assessment, prosthetics research, animal behavior, surgical evaluation, and other applications requiring calibrated multi-camera reconstruction.

Funding supports four priorities: stabilizing and documenting public APIs so researchers can depend on each package and contributors can extend them through clear protocols; defining shared data schemas, provenance records, and quality metrics between pipeline stages so outputs from any component can be consumed downstream regardless of the AI model or tracking method used; building programmatic, command-line, and batch-processing workflows for large datasets, scripted analyses, workflow engines, and agentic research environments beyond interactive GUI use; and releasing benchmarking and validation tools that let researchers evaluate reconstruction quality, compare models and pipeline configurations, and assess whether outputs suit their scientific use case.

If successful, this proposal makes the Skelly ecosystem a reusable foundation for camera-based movement analysis across the life sciences. Researchers will be able to assemble reliable pipelines from tested components rather than rebuilding capture, calibration, pose-estimation integration, 3D reconstruction, and data representation for each new project. A lab studying human gait, animal behavior, prosthetic alignment, or clinical movement can combine only the components it needs, substitute task-specific AI models, and preserve the metadata and quality metrics required for reproducible analysis.

Upstream, the project gives AI pose-estimation models a clear path into calibrated 3D research workflows — model developers can connect new trackers to shared interfaces, benchmark them, and make them available to movement researchers without rebuilding the full stack. Downstream, it produces standardized, provenance-rich outputs consumable by biomechanics tools, statistical workflows, and ML pipelines.

The work supports AI enablement and large-scale analysis by providing stable APIs, batch-processing workflows, shared data schemas, and validation protocols that allow researchers, workflow engines, and AI agents to move from raw synchronized video to structured 3D movement datasets reproducibly. The result is an ecosystem where improvements to any stage — capture, tracking, reconstruction, or analysis — propagate across projects and research communities.

## Landscape Analysis 
*We are looking for proposals from software projects with demonstrated traction and adoption. Briefly describe other software tools that the audience for this proposal primarily uses (including proprietary alternatives, if they exist), and how the software project(s) in your proposal compare in terms of user base, adoption, functionality, and maturity relative to their target audience. You can add indicators of adoption and usage as needed. Please indicate if the software is used in AI applications and workflows. (1,500 characters maximum)*

Camera-based measurement in the life sciences spans distinct communities — human motion capture (biomechanics, rehabilitation, neuroscience), animal behavior tracking (neuroscience, ecology), and emerging applications in surgical assessment and phenotyping. Each has developed tools largely in isolation.

In human motion capture: Theia3D is proprietary (~$15K–$30K, closed-source); OpenCap is open-source but locked to one backend and iOS hardware; Pose2Sim supports multiple backends but is a processing workflow, not a component library. In animal behavior: DeepLabCut and SLEAP provide pose estimation but require researchers to build their own capture, calibration, and reconstruction infrastructure. No existing ecosystem provides shared, composable libraries for the full stack — capture, calibration, pose estimation, reconstruction, and data representation — across domains.

The Skelly ecosystem is the only open-source effort providing independently installable components for each pipeline stage with standardized interfaces between them. Adoption of FreeMoCap (10,000+ users, ~140 countries, ~7,400 GitHub stars, 3,000-member community, independent publications across gait, kinematics, clinical assessment, and computational linguistics) demonstrates traction for the integrated system. The proposed work makes each library independently robust and documented for use in researcher-built pipelines and AI-driven workflows.
